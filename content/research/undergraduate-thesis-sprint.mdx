This thesis explores how large language models can adapt their pruning configuration at inference time instead of relying on one static compression profile for every prompt and every hardware condition.

## Thesis focus

The work introduces **SPRINT**: a sensitivity-guided pruning framework for inference-time adaptation of LLMs. The core idea is to decide pruning strength per prompt by combining three signals:

- oracle sensitivity labels measured from dense-versus-pruned behavior on the target backbone,
- a **Learned Complexity Router (LCR)** that predicts prompt sensitivity online,
- and a **DDQN-based controller** that reads router output together with live hardware telemetry and early backbone signals.

## Problem addressed

Large language models are powerful, but they are expensive to run on consumer or edge hardware. Static compression methods can reduce model size, yet they apply the same pruning level to every prompt regardless of prompt difficulty, latency constraints, or device state. That mismatch often wastes compute on easy prompts and harms output quality on difficult ones.

## What the thesis contributes

- A balanced **10,000-prompt oracle dataset** covering GSM8K, MBPP, WikiText-2, MMLU, and BoolQ.
- A **BERT-mini-based LCR** trained to estimate prompt sensitivity to pruning before inference.
- A **DDQN controller** that selects from a discrete action space of structural pruning options at runtime.
- A **physically correct pruning engine** that removes transformer layers to produce real latency gains rather than only weight sparsity.

## Reported outcome

Across the evaluation reported in the thesis, SPRINT achieved meaningful runtime savings while keeping quality degradation bounded. The final system demonstrated strong router correlation with oracle labels and delivered real end-to-end inference speedup by selecting structural pruning actions dynamically instead of locking the model into one static configuration.
